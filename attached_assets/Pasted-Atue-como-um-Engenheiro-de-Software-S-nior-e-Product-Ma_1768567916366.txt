Atue como um Engenheiro de Software Sênior e Product Manager. Vamos construir uma aplicação Full-Stack chamada **"MetaTask"**.

**Visão Geral do Produto:**
O MetaTask é uma plataforma de "Engenharia Reversa de Vida". Diferente de to-do lists comuns, o usuário NÃO cadastra metas. Ele passa por uma **Anamnese** (Quiz) e o sistema prescreve automaticamente uma **Trilha de Evolução (Anos 1 a 4)** com hábitos diários prontos.

**Stack Tecnológica (Replit):**
* **Frontend:** React + Vite + TailwindCSS + Lucide Icons (Estilo Apple/Clean/Glassmorphism).
* **Backend:** Node.js + Express.
* **Database:** PostgreSQL (Nativo do Replit).
* **ORM:** Drizzle ORM (para gerenciamento do schema e queries).
* **Auth:** Passport.js ou autenticação baseada em sessão simples.

---

### 1. Estrutura do Banco de Dados (Schema)
Preciso que você crie as seguintes tabelas no PostgreSQL usando Drizzle:

1.  **users**: `id`, `email`, `password_hash`, `name`, `is_premium` (boolean), `current_track_level` (1 a 4), `last_assessment_date` (timestamp).
2.  **tracks**: `id`, `level` (1, 2, 3, 4), `title` (ex: "O Arquiteto da Base"), `description`, `objective`.
3.  **track_pillars**: `id`, `track_id`, `category` (Saúde, Finanças, Mente, etc), `description` (A regra geral, ex: "Dormir 8h").
4.  **daily_tasks**: `id`, `track_pillar_id`, `title` (A ação prática, ex: "Caminhada 30min"), `frequency_per_week` (int), `is_habit` (boolean).
5.  **user_progress**: `id`, `user_id`, `task_id`, `completed_at` (date), `status` (completed).
6.  **assessment_answers**: `id`, `user_id`, `question_block`, `answer_text`, `created_at`.

### 3. Fluxo de Usuário e Anamnese (Frontend)

**Passo 1: Onboarding (A Anamnese)**
Se o usuário é novo (ou `last_assessment_date` > 6 meses), mostre um formulário estilo "Typeform" (um card por vez) com estas perguntas. Salve as respostas.

* **Bloco 1 (Biologia):**
    1. Como você se define hoje? [Sedentário, Praticante casual, Atleta consistente, Atleta elite]
    2. De 0 a 10, quanto é comida de verdade vs processados? [Slider 0-10]
    3. Sono: [Descansado/Regrado, Caótico/Cansado]
    4. Energia: [Constante, Quedas bruscas]
* **Bloco 2 (Recursos):**
    1. Finanças: [Endividado, Estável, Investidor Iniciante, Investidor Avançado]
    2. Carreira: [Satisfeito, Buscando Transição]
* **Bloco 3 (Intelecto):**
    1. Idiomas/Habilidades: [Input de texto livre]
    2. Leitura (livros/ano): [Input numérico]
* **Bloco 4 (Mente):**
    1. Saúde Mental: [Input: Como lida com estresse?]
    2. Espiritualidade: [Conexão Maior, Racional/Pragmático]
* **Bloco 5 (Conexão):**
    1. Círculo Social: [Me puxam pra cima, Me puxam pra baixo]
    2. Legado: [Input: Qual problema do mundo resolveria?]

**Lógica de Atribuição:**
Após a Anamnese, o sistema deve sugerir automaticamente a trilha. Para este MVP:
* Se resposta "Sedentário" ou "Endividado" -> Atribuir **Trilha Ano 1**.
* Caso contrário, permitir que o usuário escolha começar pelo Ano 1 (recomendado) ou Ano 2.

### 4. Dashboard Principal (A "Apple Vibe")
* **Header:** "Bom dia, [Nome]. Você está no Ano [X]: [Titulo da Trilha]".
* **Cronômetro:** Contagem regressiva para o fim do ano atual (igual ao design anterior).
* **Agenda do Dia (Hoje):** Liste as tarefas da trilha ativa.
    * Exemplo: Se o usuário está na Trilha 1, mostre "Caminhada", "Mindfulness", "Regra 80/20".
    * Ao clicar no Checkbox, salve no banco (`user_progress`) e anime uma barra de progresso.
* **Menu Lateral/Inferior:** Dashboard, Minha Evolução (Histórico), Anamnese (Visualizar respostas), Perfil (Plano Pro).
